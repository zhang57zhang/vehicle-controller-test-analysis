# 快速启动指南

## 🚀 三步快速启动

### 第一步：检查环境

打开命令提示符（CMD），输入：

```bash
python --version
node --version
```

如果显示版本号，环境OK。如果提示"不是内部或外部命令"，请先安装：
- Python：https://www.python.org/downloads/
- Node.js：https://nodejs.org/

### 第二步：一键启动

**推荐方式（无乱码）：** 双击运行 `一键启动_新版.bat` 文件

**备用方式：** 双击运行 `一键启动.bat` 文件

### 第三步：访问系统

浏览器访问：http://localhost:5173

## ❓ 遇到问题？

### 问题1：启动脚本出现乱码
**解决方案：** 使用 `一键启动_新版.bat` 替代 `一键启动.bat`

### 问题2：端口被占用
运行 `停止服务.bat`，然后重新启动

### 问题3：无法访问
检查防火墙是否阻止了Python和Node.js

### 问题4：服务窗口自动关闭
手动进入对应目录启动服务，查看错误信息

### 问题5：PowerShell执行策略错误
**解决方案：**
1. 右键点击 `一键启动_新版.bat`
2. 选择"以管理员身份运行"

### 问题6：提示"No module named 'xxx'"或类似依赖错误
**解决方案：**
1. 双击运行 `修复依赖.bat`
2. 等待依赖修复完成
3. 重新运行 `一键启动_新版.bat`

### 问题7：访问localhost:5173显示"拒绝连接"
**解决方案：**
1. **检查前端服务状态**
   - 双击运行 `测试前端连接.bat`
   - 查看前端服务是否正常启动

2. **诊断问题**
   - 双击运行 `诊断前端连接问题.bat`
   - 查看端口占用和服务状态

3. **手动启动前端**
   - 双击运行 `启动前端服务.bat`
   - 手动启动前端服务

4. **检查浏览器设置**
   - 确保浏览器访问 http://localhost:5173
   - 检查浏览器代理设置

5. **清理和重启**
   - 运行 `停止服务.bat`
   - 重启电脑后重新运行 `一键启动_新版.bat`

## 🔧 依赖修复

**说明：** 如果遇到依赖问题（如找不到模块、导入错误等），可以使用修复脚本。

**使用方法：**
1. 双击运行 `修复依赖.bat`
2. 脚本会自动：
   - 删除旧的虚拟环境
   - 创建新的虚拟环境
   - 安装所有依赖
   - 验证关键包
3. 修复完成后，重新运行启动脚本

**适用场景：**
- 提示 "No module named 'pydantic_settings'"
- 提示 "No module named 'xxx'"
- 虚拟环境损坏
- 依赖版本冲突

## 🔧 前端连接问题修复

**说明：** 如果前端页面无法访问，可以使用诊断和修复工具。

### 解决方案

1. **测试连接**
   - 双击运行 `测试前端连接.bat`
   - 检查前端服务是否正常运行

2. **诊断问题**
   - 双击运行 `诊断前端连接问题.bat`
   - 自动检测端口占用、服务状态、目录结构
   - 提供详细的错误信息

3. **手动启动**
   - 双击运行 `启动前端服务.bat`
   - 手动启动前端开发服务器

### 常见问题

**问题：端口5173被占用**
- **原因：** 其他程序占用了端口
- **解决：** 关闭其他程序或修改端口

**问题：前端服务启动失败**
- **原因：** 依赖缺失或代码错误
- **解决：** 运行 `修复依赖.bat` 后重试

**问题：页面空白或错误**
- **原因：** 前端代码编译错误
- **解决：** 查看浏览器控制台错误信息
