# 乱码问题解决方案

## 问题说明

Windows批处理文件（.bat）在某些情况下会出现中文乱码，这是由于：

1. **编码问题：** Windows默认使用GBK/GB2312编码，而编辑器可能使用UTF-8编码
2. **字体问题：** 部分字体不支持中文显示
3. **系统设置：** Windows区域和语言设置影响编码

---

## 解决方案

### 方案一：使用PowerShell版本（推荐）

**文件：** `一键启动_新版.bat`

**原理：**
- 使用PowerShell执行脚本
- PowerShell原生支持UTF-8
- 完全避免编码问题

**使用方法：**
1. 双击运行 `一键启动_新版.bat`
2. 如果提示执行策略错误，右键选择"以管理员身份运行"

**优点：**
- ✅ 完全避免乱码
- ✅ 彩色输出，界面美观
- ✅ 更详细的错误提示

---

**文件：** `一键启动.bat`（已更新为英文版本）

**原理：**
- 使用英文替代中文
- 避免中文编码问题

**使用方法：**
双击运行 `一键启动.bat`

**优点：**

**步骤：**

## 常见问题

### Q: 为什么PowerShell版本会出现执行策略错误？

**A:** PowerShell默认限制脚本执行。解决方法：
1. 右键点击 `一键启动_新版.bat`
2. 选择"以管理员身份运行"

或在PowerShell中运行：
```powershell
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
```

### Q: 英文版本的脚本可以理解吗？

**A:** 英文版本非常简单，主要包含：
- Check：检查
- Start：启动
- OK/Info/Warning/Error：提示信息
- Backend/Frontend：后端/前端

### Q: 如果两个脚本都无法运行怎么办？

**A:** 手动启动服务：

**启动后端：**
```bash
cd backend
python -m venv venv
venv\Scripts\activate.bat
pip install -r requirements.txt
uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
```

**启动前端（新窗口）：**
```bash
cd frontend
npm install
npm run dev
```

## 技术细节

### 批处理文件编码问题

批处理文件默认使用ANSI编码（GBK），而现代编辑器默认使用UTF-8编码。

**解决方案：**
1. 保存时使用ANSI编码
2. 或使用英文替代中文
3. 或使用PowerShell替代批处理

### PowerShell执行策略

PowerShell为了安全，默认不允许执行脚本。

**解决方法：**
1. 在批处理文件中使用 `-ExecutionPolicy Bypass`
2. 或使用 `RemoteSigned` 策略
