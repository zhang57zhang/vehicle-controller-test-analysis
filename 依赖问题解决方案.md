# 依赖问题解决方案

## 问题描述

在运行 `一键启动.bat` 时，可能会遇到类似以下错误：

```
ModuleNotFoundError: No module named 'pydantic_settings'
```

或其他类似错误：
```
ModuleNotFoundError: No module named 'xxx'
```

---

## 原因分析

### 1. 虚拟环境未正确创建

**原因：**
- 虚拟环境创建失败但没有报错
- 使用了错误的Python版本
- 创建过程中被中断

### 2. 依赖安装失败

**原因：**
- 网络问题导致下载失败
- pip版本过旧
- 某个包安装失败但未报错
- 权限问题

### 3. 使用了错误的Python环境

**原因：**
- 激活了虚拟环境但实际使用了系统Python
- 多个Python版本混淆
- 环境变量问题

---

## 解决方案

### 方案一：使用依赖修复脚本（推荐）

**特点：**
- ✅ 完全自动化

### 方案二：手动修复（高级用户）

#### 步骤1：删除旧虚拟环境

打开命令提示符（CMD），进入backend目录：

```bash
cd vehicle-controller-test-analysis\backend
rmdir /s /q venv
```

#### 步骤2：创建新虚拟环境

```bash
python -m venv venv
```

#### 步骤3：激活虚拟环境

**Windows CMD:**
```bash
venv\Scripts\activate.bat
```

**Windows PowerShell:**
```powershell
venv\Scripts\Activate.ps1
```

#### 步骤4：升级pip

```bash
python -m pip install --upgrade pip
```

#### 步骤5：安装依赖

```bash
pip install -r requirements.txt
```

#### 步骤6：验证安装

```bash
python -c "import pydantic_settings; print(pydantic_settings.__version__)"
```

如果输出版本号，说明安装成功。

#### 步骤7：启动服务

```bash
uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
```

### 方案三：检查Python版本兼容性

**要求：** Python 3.9 或更高版本

**检查当前版本：**

```bash
python --version
```

**如果版本过低：**
1. 下载并安装最新的Python 3.10或3.11
2. 下载地址：https://www.python.org/downloads/
3. 安装时记得勾选 "Add Python to PATH"
4. 安装完成后重新运行 `修复依赖.bat`

## 常见依赖问题

### 问题1：pydantic_settings 未找到

**原因：**
- pydantic-settings v2.x改变了导入方式
- 旧代码可能使用 `from pydantic import BaseSettings`
- 新代码使用 `from pydantic_settings import BaseSettings`

**解决方案：**
运行 `修复依赖.bat`，它会自动安装正确版本的pydantic-settings。

---

### 问题2：fastapi 相关错误

**原因：**
- FastAPI版本过旧
- Starlette版本不匹配

**解决方案：**
运行 `修复依赖.bat`，它会安装兼容的版本。

---

### 问题3：pandas/numpy 相关错误

**原因：**
- numpy版本冲突
- pandas和numpy版本不兼容

**解决方案：**
运行 `修复依赖.bat`，它会安装兼容的版本。

---

### 问题4：网络问题导致安装失败

**原因：**
- 网络连接不稳定
- pip源访问慢

**解决方案：**

**使用国内镜像源：**

```bash
pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple
```

**或配置永久镜像源：**

创建文件 `pip.ini`，内容：

```ini
[global]
index-url = https://pypi.tuna.tsinghua.edu.cn/simple
```

## 验证安装

运行 `修复依赖.bat` 后，会自动验证关键包。也可以手动验证：

```bash
# 进入backend目录
cd vehicle-controller-test-analysis\backend

# 激活虚拟环境
venv\Scripts\activate.bat

# 验证包
python -c "import pydantic_settings; print('OK')"
python -c "import fastapi; print('OK')"
python -c "import uvicorn; print('OK')"
python -c "import sqlalchemy; print('OK')"
python -c "import pandas; print('OK')"
```

如果所有命令都输出 "OK"，说明依赖安装正确。

## 预防措施

### 1. 定期更新依赖

```bash
cd backend
venv\Scripts\activate.bat
pip install --upgrade -r requirements.txt
```

### 2. 避免混合使用虚拟环境和系统Python

始终在虚拟环境中运行：
```bash
venv\Scripts\activate.bat
```

### 3. 不要手动修改虚拟环境

虚拟环境应该只通过 `pip install` 修改，不要手动复制文件。

## 技术细节

### 为什么会出现"No module named"错误？

**Python模块搜索顺序：**
1. 当前目录
2. PYTHONPATH环境变量
3. 标准库目录
4. site-packages目录（虚拟环境中的包安装位置）

如果使用了错误的Python环境，或者虚拟环境中的包未正确安装，就会出现"No module named"错误。

### 为什么需要删除旧虚拟环境？

**问题：**
- 旧的虚拟环境可能损坏
- 包之间可能存在版本冲突
- 删除后重新创建可以确保环境干净

**注意：**
删除虚拟环境后，需要重新安装所有依赖。
