# 车载控制器测试数据分析系统 - Windows启动指南

## 📋 概述
本系统提供了优化的Windows批处理(.bat)启动脚本，不再依赖PowerShell(.ps1)文件，提供更好的兼容性和用户体验。

## 🚀 启动脚本

### 1. 前端启动
**文件：** `前端启动.bat`

**功能：**
- 自动停止现有服务
- 检查前端目录完整性
- 自动安装依赖（如果需要）
- 启动前端开发服务器
- 验证服务状态
- 提供访问地址和故障排除建议

**使用方法：**
1. 双击 `前端启动.bat`
2. 等待脚本执行完成
3. 在浏览器中访问 http://localhost:5173

### 2. 停止服务
**文件：** `停止服务.bat`

**功能：**
- 停止Node.js和Python进程
- 检查端口释放状态
- 提供重启建议

**使用方法：**
1. 双击 `停止服务.bat`
2. 等待脚本执行完成

## 🎯 优化特性

### ✅ 改进内容
1. **纯批处理实现** - 不再依赖PowerShell，提升兼容性
2. **UTF-8编码支持** - 使用`chcp 65001`支持中文显示
3. **智能依赖检查** - 自动检测并安装缺失的node_modules
4. **服务状态验证** - 检查端口占用情况
5. **友好的错误处理** - 提供清晰的错误提示
6. **窗口管理** - 在新窗口中运行服务，不影响主窗口

### 🛠️ 技术细节
- 编码：UTF-8 (chcp 65001)
- 端口：5173 (前端)
- 服务管理：taskkill + netstat
- 依赖检查：package.json + node_modules目录

## 🔧 故障排除

### 常见问题
1. **端口被占用**
   - 运行 `停止服务.bat` 释放端口
   - 检查是否有其他服务占用5173端口

2. **依赖缺失**
   - 脚本会自动安装npm依赖
   - 如果失败，可手动在frontend目录运行 `npm install`

3. **权限问题**
   - 确保脚本有执行权限
   - 可能需要以管理员身份运行

4. **显示问题**
   - 如果中文显示异常，确认系统支持UTF-8
   - 可尝试设置系统区域编码为中文

### 验证服务状态
启动后可通过以下方式验证：
- 访问 http://localhost:5173
- 运行 `netstat -an | findstr :5173` 检查端口
- 查看新的命令提示符窗口输出

## 📝 使用建议
1. **首次使用前**：确保已安装Node.js和npm
2. **日常使用**：直接双击 `前端启动.bat` 即可
3. **问题排查**：先运行 `停止服务.bat` 清理环境
4. **系统重启后**：需要重新启动服务
