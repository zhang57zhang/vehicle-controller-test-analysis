## 系统简介

车载控制器测试数据分析系统是一个用于新能源汽车车载控制器测试数据管理和分析的平台，实现测试数据的统一导入、自动化分析、异常检测和标准化报告生成。

**主要功能：**
- 📁 项目管理：创建、编辑、删除测试项目
- 📤 数据导入：支持多种格式文件上传（CSV、Excel、MAT、DBC、CAN log等）
- 🔧 DBC文件管理：上传和管理DBC通信协议文件
- 📊 数据分析：信号映射配置、自定义信号、分析参数配置
- 📈 可视化展示：测试数据和结果的可视化展示

## 快速开始

### 方式一：一键启动（推荐）

**前提条件：** Python和Node.js已安装

1. 双击运行 `一键启动.bat` 文件
2. 等待服务启动完成
3. 浏览器访问 `http://localhost:5173`
4. 开始使用系统

### 方式二：手动启动

#### 启动后端服务

1. 打开命令提示符，进入 `backend` 目录：

```bash
cd vehicle-controller-test-analysis\backend
```

2. 创建虚拟环境（首次运行）：

```bash
python -m venv venv
```

3. 激活虚拟环境：

```bash
venv\Scripts\activate.bat
```

4. 安装依赖（首次运行）：

```bash
pip install -r requirements.txt
```

5. 启动后端服务：

```bash
uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
```

后端服务启动成功后，会显示：

```
INFO:     Uvicorn running on http://0.0.0.0:8000
INFO:     Application startup complete.
```

#### 启动前端服务

1. 打开新的命令提示符窗口，进入 `frontend` 目录：

```bash
cd vehicle-controller-test-analysis\frontend
```

2. 安装依赖（首次运行）：

```bash
npm install
```

3. 启动前端服务：

```bash
npm run dev
```

前端服务启动成功后，会显示：

```
VITE v5.4.21 ready in XXX ms

➜  Local:   http://localhost:5173/
➜  Network: http://192.168.x.x:5173/
```

### 访问系统

服务启动成功后，在浏览器中访问：

- **前端界面：** http://localhost:5173
- **后端API：** http://localhost:8000
- **API文档：** http://localhost:8000/api/docs

## 功能介绍

### 1. 项目管理页面

**访问路径：** `/`

**功能描述：** 管理和配置测试项目

**主要功能：**

#### 创建项目
1. 点击右上角"新建项目"按钮
2. 输入项目名称（必填）
3. 输入项目描述（可选）
4. 点击"创建"按钮

#### 编辑项目
1. 在项目列表中找到要编辑的项目
2. 点击操作列中的"编辑"按钮
3. 修改项目信息
4. 点击"保存"按钮

#### 删除项目
1. 在项目列表中找到要删除的项目
2. 点击操作列中的"删除"按钮
3. 确认删除操作

#### 查看项目详情
1. 在项目列表中找到要查看的项目
2. 点击操作列中的"查看"按钮
3. 查看项目详细信息

#### 项目列表显示

- **项目名称**：项目的名称
- **描述**：项目的描述信息
- **DBC文件**：显示是否已配置DBC文件（已配置/未配置）
- **创建时间**：项目的创建时间
- **更新时间**：项目的最后更新时间
- **操作**：查看、编辑、删除按钮

---

### 2. 数据导入页面

**访问路径：** `/import`

**功能描述：** 上传测试数据、DBC文件和测试用例

**前提条件：** 需要先创建或选择一个项目

**主要功能：**

#### 上传测试数据

1. 选择数据类型：
   - MIL（模型在环测试）
   - HIL（硬件在环测试）
   - DVP（设计验证）
   - VEHICLE（整车测试）
   - MANUAL（手动测试）
   - AUTO（自动测试）

2. 上传文件：
   - 点击上传区域选择文件
   - 或直接拖拽文件到上传区域
   - 支持格式：`.mat`, `.csv`, `.xlsx`, `.log`, `.blf`, `.asc`, `.xml`, `.json`

3. 查看上传进度：
   - 实时显示上传进度条
   - 显示上传百分比

#### 上传DBC文件

1. 点击DBC文件上传区域
2. 选择DBC文件
3. 或直接拖拽DBC文件到上传区域
4. 支持格式：`.dbc`, `.arxml`, `.xml`

#### 导入测试用例

1. 点击测试用例上传区域
2. 选择Excel文件
3. 或直接拖拽Excel文件到上传区域
4. 支持格式：`.xlsx`, `.xls`

#### 查看已上传文件

- **测试数据列表**：显示所有已上传的测试数据文件
  - 文件名、格式、类型、大小
  - 删除操作

- **DBC文件列表**：显示所有已上传的DBC文件
  - 文件名、版本
  - 删除操作

---

### 3. 数据分析页面

**访问路径：** `/analysis`

**功能描述：** 配置分析参数并执行数据分析

**前提条件：** 需要先上传测试数据

**主要功能：**

#### 分析配置标签

**选择测试数据：**
1. 从下拉列表中选择要分析的测试数据文件
2. 列表显示文件名、格式、数据类型

**配置分析参数：**
- **采样率**：设置目标采样率（Hz）
- **插值方法**：选择插值方式
  - 线性插值
  - 样条插值
  - 阶梯插值

**执行分析：**
1. 点击"开始分析"按钮
2. 等待分析完成
3. 查看分析结果

#### 信号映射标签

**功能描述：** 配置DBC信号与数据源信号的映射关系

**添加信号映射：**
1. 点击"添加映射"按钮
2. 填写映射信息：
   - 信号别名（必填）
   - DBC信号（可选）
   - 数据源信号（可选）
   - 原始单位（可选）
   - 目标单位（可选）
   - 转换公式（可选）
   - 描述（可选）
3. 点击"添加"按钮

**编辑信号映射：**
1. 在列表中找到要编辑的映射
2. 点击"编辑"按钮
3. 修改映射信息
4. 点击"保存"按钮

**删除信号映射：**
1. 在列表中找到要删除的映射
2. 点击"删除"按钮
3. 确认删除操作

#### 自定义信号标签

**功能描述：** 创建基于计算的自定义信号

**添加自定义信号：**
1. 点击"添加信号"按钮
2. 填写信号信息：
   - 信号别名（必填）
   - 计算公式（必填）
   - 输入信号（用逗号分隔）
   - 单位（可选）
   - 描述（可选）
3. 点击"添加"按钮

**编辑自定义信号：**
1. 在列表中找到要编辑的信号
2. 点击"编辑"按钮
3. 修改信号信息
4. 点击"保存"按钮

**删除自定义信号：**
1. 在列表中找到要删除的信号
2. 点击"删除"按钮
3. 确认删除操作

## 常见问题

### 1. 启动问题

#### Q: 双击"一键启动.bat"后，提示"未检测到Python"或"未检测到Node.js"

**A:** 请先安装Python和Node.js：
- Python下载：https://www.python.org/downloads/
- Node.js下载：https://nodejs.org/

安装时记得勾选"Add Python to PATH"或"Add to PATH"选项。

#### Q: 后端服务启动失败，提示"端口8000已被占用"

**A:** 可能是之前的后端服务没有正常关闭。解决方法：
1. 运行"停止服务.bat"
2. 或手动关闭所有命令提示符窗口
3. 重新启动服务

#### Q: 前端服务启动失败，提示"端口5173已被占用"

**A:** 可能是之前的前端服务没有正常关闭。解决方法：
1. 运行"停止服务.bat"
2. 或手动关闭所有命令提示符窗口
3. 重新启动服务

#### Q: 服务启动成功，但浏览器无法访问

**A:** 可能是防火墙阻止了访问。解决方法：
1. 检查Windows防火墙设置
2. 允许Python和Node.js通过防火墙
3. 或暂时关闭防火墙测试

---

### 2. 使用问题

#### Q: 创建项目时提示"请输入项目名称"

**A:** 项目名称是必填项，请确保输入了项目名称。

#### Q: 上传文件时提示"文件格式不支持"

**A:** 请检查文件格式是否在支持列表中：
- 测试数据：`.mat`, `.csv`, `.xlsx`, `.log`, `.blf`, `.asc`, `.xml`, `.json`
- DBC文件：`.dbc`, `.arxml`, `.xml`
- 测试用例：`.xlsx`, `.xls`

#### Q: 上传大文件时很慢或失败

**A:** 可能是网络或服务器性能问题。解决方法：
1. 检查文件大小是否过大（建议小于500MB）
2. 检查网络连接是否正常
3. 检查后端服务是否正常运行

#### Q: 数据分析时提示"请选择测试数据"

**A:** 需要先在数据导入页面上传测试数据文件。

### 3. 技术问题

#### Q: 如何查看系统日志？

**A:** 系统日志在服务启动的命令提示符窗口中查看：
- 后端日志：在"车载控制器测试-后端"窗口中
- 前端日志：在"车载控制器测试-前端"窗口中

#### Q: 如何重新安装依赖？

**A:**

**后端依赖：**
```bash
cd backend
rmdir /s /q venv
python -m venv venv
venv\Scripts\activate.bat
pip install -r requirements.txt
```

**前端依赖：**
```bash
cd frontend
rmdir /s /q node_modules
del package-lock.json
npm install
```

#### Q: 如何更新系统到最新版本？

**A:** 如果使用Git版本控制：
```bash
git pull origin master
```

然后按照"快速开始"部分的说明重新安装依赖（如果需要）。
