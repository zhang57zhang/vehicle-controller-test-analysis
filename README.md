# 车载控制器测试数据分析与测试报告编写系统

## 📋 项目概述

这是一个用于新能源汽车车载控制器测试数据管理和分析的平台，实现测试数据的统一导入、自动化分析、异常检测和标准化报告生成。

**状态:** 🟢 开发中 - Phase 1 Week 1
**开始日期:** 2026-03-17
**目标完成:** 2026-05-17（约8周）
**当前进度:** Phase 1 Week 1 (60%)

## 🎯 核心目标

- **统一管理**: 测试数据集中存储和管理
- **自动分析**: 自动化数据分析和异常检测
- **双报告**: 同时生成标准报告（对外）和溯源报告（内部审核）
- **信号映射**: 支持不同车企项目的信号归一化
- **本地部署**: 完全本地化，不出内网，数据安全

## 🎯 部署方式

**关键约束：**
- ✅ 本地部署（不出本地网络）
- ✅ 无云端依赖
- ✅ 支持离线分析
- ✅ Web界面 + 本地应用程序双端支持

## 📊 支持的测试类型

| 测试类型 | 数据格式 | 分析重点 |
|---------|---------|---------|
| MIL (模型在环) | .mat, .csv | 逻辑正确性、算法验证 |
| HIL (硬件在环) | .mat, .log, .blf, .asc | 实时性、精度、边界条件 |
| DVP (设计验证) | .csv, .xlsx, .dat | 性能指标、耐久性 |
| 整车测试 | .log, .asc, .blf | 功能完整性、用户体验 |
| 手动测试 | .xlsx, .csv | 用例执行、定性评价 |
| 自动测试 | .xml, .json, .csv | 回归测试、覆盖率 |

## 📈 数据规模

**单次测试数据量：** 2KB ~ 500MB

| 规模 | 大小 | 处理策略 |
|------|------|---------|
| 小型 | 2KB ~ 1MB | 内存直接处理 |
| 中型 | 1MB ~ 50MB | 内存+缓存 |
| 大型 | 50MB ~ 200MB | 流式处理 |
| 超大型 | 200MB ~ 500MB | 分块+流式 |
