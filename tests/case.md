# 车辆上电启动及运行测试用例

## 文档信息
- **测试项目**: VCU/MCU/BMS系统集成测试
- **测试类型**: 功能测试、时序测试
- **创建日期**: 2026-03-27
- **文档版本**: V1.0

---

## 测试目的
验证车辆上电启动后，VCU自检、MCU与BMS协调控制加速减速、以及DCDC补电功能的时序和功能是否符合设计要求。

---

## 测试环境

### 硬件要求
- 整车控制器 (VCU) x1
- 电机控制器 (MCU) x1
- 电池管理系统 (BMS) x1
- DCDC转换器 x1
- 低压蓄电池 (12V)
- 高压动力电池组
- CAN总线分析仪
- 示波器（可选）

### 软件要求
- CAN监控软件
- VCU诊断工具
- 车辆仿真平台或实车环境

### 前提条件
- 各控制器软件版本已确认
- 高压电池SOC > 20%
- 低压蓄电池电压正常 (11.5V - 14.5V)
- 车辆处于静止状态
- 环境温度: -20°C ~ 55°C

## 测试用例

### TC-01: 车辆上电启动测试

| 项目 | 内容 |
|------|------|
| **测试ID** | TC-01 |
| **测试名称** | 车辆上电启动及VCU自检 |
| **优先级** | P0 (高) |
| **前置条件** | 车辆处于OFF状态，钥匙在OFF位置或按钮未按下 |
| **测试步骤** | 1. 将钥匙旋至ON位置或按下启动按钮<br>2. 监测VCU自检信号<br>3. 检查各控制器状态字<br>4. 观察仪表盘指示灯 |
| **预期结果** | 1. VCU接收上电指令，进入初始化状态<br>2. VCU自检完成，状态字无故障<br>3. MCU、BMS进入待机状态<br>4. 仪表盘READY灯点亮（或设计指示灯）<br>5. 无故障码存储 |
| **测试数据** | - VCU状态字: 0xXXXX (无故障)<br>- 自检完成时间: < 3s |
| **备注** | 记录上电过程中的CAN报文时序 |

### TC-02: 加速控制测试

| 项目 | 内容 |
|------|------|
| **测试ID** | TC-02 |
| **测试名称** | VCU/MCU/BMS协调加速测试 |
| **优先级** | P0 (高) |
| **前置条件** | 车辆READY状态，档位在D档，手刹已释放 |
| **测试步骤** | 1. 缓踩油门踏板至10%开度<br>2. 监测MCU扭矩响应<br>3. 检查BMS电流输出<br>4. 监测VCU控制策略<br>5. 逐步增加踏板至50%、80%、100% |
| **预期结果** | 1. MCU接收VCU扭矩指令，响应时间< 100ms<br>2. BMS正常放电，电流与扭矩匹配<br>3. VCU根据踏板开度输出对应扭矩<br>4. 无过流、过压保护触发<br>5. 加速曲线平滑，无抖动 |
| **测试数据** | - 油门踏板开度 vs 扭矩输出<br>- 响应延迟时间<br>- 电池放电电流 (A) |
| **备注** | 记录每个踏板开度下的实际扭矩值 |

### TC-03: 减速控制测试

| 项目 | 内容 |
|------|------|
| **测试ID** | TC-03 |
| **测试名称** | VCU/MCU/BMS协调减速测试 |
| **优先级** | P0 (高) |
| **前置条件** | 车辆处于行驶状态，车速 > 20km/h |
| **测试步骤** | 1. 缓松油门踏板至0%<br>2. 监测能量回收状态<br>3. 检查BMS充电电流<br>4. 观察减速减速度<br>5. 紧急制动时踩下刹车踏板 |
| **预期结果** | 1. 松开油门后，MCU进入能量回收模式<br>2. BMS接收充电电流，在安全范围内<br>3. 减速度平滑，无明显顿挫<br>4. 制动优先级: 机械制动 > 能量回收<br>5. 无过充保护触发 |
| **测试数据** | - 能量回收电流 (A)<br>- 减速度 (m/s²)<br>- 回收效率 (%) |
| **备注** | 测试不同车速下的能量回收策略 |

### TC-04: 驾驶2分钟后DCDC启动测试

| 项目 | 内容 |
|------|------|
| **测试ID** | TC-04 |
| **测试名称** | 驾驶2分钟后DCDC启动及补电功能测试 |
| **优先级** | P1 (中) |
| **前置条件** | 车辆READY，正常行驶中 |
| **测试步骤** | 1. 开始计时，车辆正常行驶<br>2. 监测低压蓄电池电压<br>3. 记录DCDC启动时刻<br>4. 监测DCDC输出电压和电流<br>5. 持续监测2分钟后补电状态 |
| **预期结果** | 1. 驾驶2分钟后（或低压电压低于阈值时），DCDC自动启动<br>2. DCDC输出电压稳定在13.5V - 14.5V<br>3. 低压蓄电池电压逐渐回升<br>4. DCDC工作无异常声音或过热<br>5. 补电电流在额定范围内 (通常 < 80A) |
| **测试数据** | - DCDC启动时间: 120s ± 5s<br>- 启动前低压电压 (V)<br>- 启动后低压电压 (V)<br>- DCDC输出电流 (A) |
| **备注** | 验证DCDC启动条件（时间/电压阈值） |

## 时序要求

### 上电启动时序
```
T0:      用户操作（上电）
T0+10ms: VCU接收上电信号
T0+50ms: VCU发送初始化指令给MCU、BMS
T0+200ms: MCU、BMS响应进入待机状态
T0+500ms: VCU完成自检
T0+2s:   READY信号输出，车辆可行驶
```

### DCDC启动时序
```
T0:      车辆开始行驶
T0+120s: DCDC启动条件满足
T0+122s: VCU发送DCDC使能指令
T0+125s: DCDC开始输出
T0+130s: 输出电压稳定在目标范围
```

## 注意事项

1. **安全第一**: 测试过程中注意高压安全，严格遵守操作规范
2. **数据记录**: 所有测试需保存完整的CAN总线日志
3. **环境控制**: 尽量在稳定环境下测试，减少外界干扰

5. **异常处理**: 发现异常立即停止测试，记录现象

## 附录

### 关键信号定义

| 信号名称 | CAN ID | 字节位 | 单位 | 备注 |
|----------|--------|--------|------|------|
| VCU状态 | 0x18FF0001 | Byte 0 | - | 0x00=OFF, 0x01=READY |
| 扭矩请求 | 0x0CF00400 | Byte 0-1 | Nm | signed, 0-2000 |
| 踏板开度 | 0x0CF00400 | Byte 2 | % | 0-100 |
| BMS电流 | 0x18FF0501 | Byte 0-1 | A | signed, 正=放电 |
| 低压电压 | 0x18FF0601 | Byte 0-1 | 0.1V | |
| DCDC状态 | 0x18FF0701 | Byte 0 | - | 0x00=OFF, 0x01=ON |
| DCDC电流 | 0x18FF0701 | Byte 1-2 | A | |

### 术语表

- **VCU**: Vehicle Control Unit (整车控制器)
- **MCU**: Motor Control Unit (电机控制器)
- **BMS**: Battery Management System (电池管理系统)
- **DCDC**: DC-DC Converter (直流转换器)
- **SOC**: State of Charge (荷电状态)
- **READY**: 车辆就绪状态，可正常行驶
