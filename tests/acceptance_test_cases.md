**优先级**: 高

- 消息ID 200 (BMS_Data) 被正确识别

**验收标准**:
✅ 文件上传成功率 100%

### 2. 数据解析准确性测试用例

#### TC-101: DBC消息解析准确性
**优先级**: 高
**描述**: 验证DBC文件中的消息定义被准确解析

**测试数据**:
```
BO_ 100 VCU_Status: 8 VCU
 SG_ VCU_State : 0|8@1+ (0.1,0) [0|25.5] "V"
```

**验证点**:
- 消息ID: 100
- 消息名称: VCU_Status
- 数据长度: 8字节
- 发送节点: VCU
- 信号名称: VCU_State
- 起始位: 0
- 信号长度: 8位
- 字节顺序: Motorola
- 数据类型: 无符号整数
- 因子: 0.1
- 偏移: 0
- 最小值: 0
- 最大值: 25.5

**验收标准**:
✅ 所有参数解析准确率 100%

---

#### TC-102: CAN信号解码准确性
**优先级**: 高
**描述**: 验证CAN原始数据能够根据DBC定义准确解码为物理值

**测试场景**:
- 原始数据: [0x02, 0x7D, 0x23, 0x00, 0x00, 0x00, 0x00, 0x00]
- 信号: VCU_State, 起始位0, 长度8, 因子0.1, 偏移0
- 期望结果: VCU_State = 0.2 * 10 = 2.0

**验收标准**:
✅ 解码准确率 >= 99.9%
✅ 精度误差 < 0.01

---

#### TC-103: 数值类型转换准确性
**优先级**: 高
**描述**: 验证不同数值类型（有符号/无符号、浮点数）的转换准确性

**测试场景**:
1. 无符号8位整数: 0-255
2. 有符号16位整数: -32768 到 32767
3. 浮点数: IEEE 754格式
4. 双精度浮点数: IEEE 754格式

**验收标准**:
✅ 所有类型转换准确率 100%

---

#### TC-104: 时间戳解析准确性
**优先级**: 高
**描述**: 验证不同时间戳格式的准确解析

**测试格式**:
- 绝对时间戳: 2024-03-17 10:00:00.000
- 相对时间戳: 0.000, 0.050, 0.100 (秒)
- CANoe时间戳格式

**验收标准**:
✅ 时间戳解析准确率 100%
✅ 时区处理正确
